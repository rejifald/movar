# Movar — source build instructions (Firefox AMO)

This document is provided to Firefox AMO reviewers per the
"reviewable sources for bundled/minified extensions" policy. It describes
how to reproduce the submitted `.zip` from source on a clean machine.

## Build environment

- **OS:** any (verified on macOS 14+, Linux x86_64). Windows untested but
  expected to work via WSL.
- **Node.js:** ≥ 22 (see `engines.node` in the repo-root `package.json`).
- **pnpm:** ≥ 11 (see `engines.pnpm` in the repo-root `package.json`).
- **Network:** required for the initial `pnpm install` to fetch packages.
  No network is used at build time.

## Source layout

- Repository root: monorepo managed with **pnpm workspaces** + **Nx**.
- Extension source: `apps/extension/src/` (TypeScript + React 19, JSX).
- Internal workspace dependencies used by the extension build:
  - `packages/lang-detect` — language detection helpers.
  - `packages/rules` — `declarativeNetRequest` rule generation.
  - `packages/shared` — shared types and storage helpers.
- Build framework: [WXT](https://wxt.dev) (`wxt.config.ts` is the manifest source of truth).

## Reproduce the AMO artifact

From the repository root:

```bash
pnpm install
pnpm --filter @movar/extension build:firefox
pnpm --filter @movar/extension zip:firefox
```

- Unpacked output: `apps/extension/.output/firefox-mv3/`
- AMO-ready zip: `apps/extension/.output/firefox-mv3-<version>.zip`
- Source bundle (if AMO requests one separately):
  `apps/extension/.output/firefox-mv3-<version>-sources.zip`
  (produced by `wxt zip` alongside the artifact).

## Notes for reviewers

- No code is fetched at runtime; the extension bundles all its dependencies.
- No remote analytics, telemetry, or eval of remote code.
- The content script in `apps/extension/src/entrypoints/content.ts` modifies
  page DOM only; outgoing-request rewrites are declared statically via
  `declarativeNetRequest` rules generated by `@movar/rules`.
- The build is deterministic given the same `pnpm-lock.yaml` and Node version.
