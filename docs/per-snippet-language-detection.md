# Per-snippet language detection: set-difference ladder + franc shadow oracle

## Relationship to the page-language ADR

[on-device-language-detection.md](./on-device-language-detection.md) replaced the **page-language** body-text fallback with the `chrome-ai → franc-min` engine roster, deliberately leaving the **per-snippet** path on `detectCyrillicLanguage` (its _Out of scope_: "stays … for short / mixed-script accuracy AND per-applyOnce performance"). This ADR **supersedes that item** — not by swapping in franc-min, but by generalizing the letter heuristic into a declarative set-difference classifier and using franc only as a gated rung-3 backstop and an off-path verifier. Rungs 1–2 stay the sole sync authority for the common case.

This ADR does not touch the engine roster, orchestrator, sampler, or page-language path.

## Implementation status (2026-06-03)

Phases 1–2 have landed in `@movar/lang-detect` and `apps/extension`:

- **Phase 1** — `LanguageProfile` (`{ code, alphabet, words: { function, frequent } }`), `classifyBySnippet` (rungs 1/2a/2b), and `uk`/`ru`/`be`/`en` profiles. `words.frequent` is generated by [`scripts/gen-word-profiles.mts`](../packages/lang-detect/scripts/gen-word-profiles.mts) (OpenSubtitles register); `be` is hand-curated as it has no corpus coverage.
- **Phase 2** — `applyContentFilter` now takes `{ candidates, enabled }` and uses `classifyBySnippet` + the per-rung `minHideMargin` allowlist predicate; `content.ts` builds `candidates = getProfiles(priority ∪ blocked)` and `enabled = new Set(priority)`.
- **Added: dominant-script scoping.** `classifyBySnippet` first restricts candidates to the text's dominant script, so a minority-script token (a Latin brand like "YouTube" in a Cyrillic title) can't tip the verdict. Surfaced by Phase-2 tests; matches the corpus fixtures' "minority script tokens shouldn't tip" intent.

**Deviation — `IMPOSED_OVER` / `deriveBlocked` deferred.** The repo already ships a _user-editable_ blocklist (`apps/extension/src/entrypoints/options/BlockedSection.tsx`, with `ru` locked). Auto-deriving `blocked` from `priority` would silently remove that UX, so Phase 2 treats the existing user-editable `blocked` as the imposed overlay: `candidates = priority ∪ blocked`, `enabled = priority`, conceal iff detected ∉ `enabled` — mathematically "hide iff the card reads as a blocked language", now via the set-difference classifier. Replacing the manual blocklist with a derived `IMPOSED_OVER` overlay is a separate UX decision (Open questions).

## Context

The per-snippet content filter is the layer that actually hides content. [`applyContentFilter`](../apps/extension/src/lib/page-content/conceal.ts) iterates `PageContentModel.nodes` and calls [`detectCyrillicLanguage(node.text)`](../apps/extension/src/lib/page-content/conceal.ts:188) per node, gated today by a hard `ru`-only check ([conceal.ts:176](../apps/extension/src/lib/page-content/conceal.ts:176)).

Two facts shape every decision:

- **Conceal is a feed/results filter, not a page-text filter.** [`buildModelForHost`](../apps/extension/src/lib/page-content/registry.ts:41) returns `null` when no extractor matches, there is **no generic extractor** (only Google + YouTube), and every `CardKind` is a feed unit ([types.ts:23](../apps/extension/src/lib/page-content/types.ts)). Conceal only ever hides structured cards — which is what makes an allowlist hide-model safe.
- [`detectCyrillicLanguage`](../packages/lang-detect/src/index.ts:48) is accurate but Cyrillic-only, hard-wired, and carries a **frequency-blind fallback** ("Cyrillic + no distinctive letters → `ru`") that both over-hides (a distinctive-free Ukrainian title like "Нова музика" is classified `ru`) and cannot scale (it can't separate nested-alphabet pairs at all).

### Why franc-min isn't the per-snippet authority

Short text (franc returns `und` below [`MIN_LENGTH = 10`](../packages/lang-detect/src/engines/franc-min.ts:24); shaky well above); false positives on native content are unacceptable; coverage gaps (~45 mapped codes, [franc-min.ts:31](../packages/lang-detect/src/engines/franc-min.ts:31)); hot-path cost. So franc is the **rung-3 backstop** and an **off-path oracle** — never the primary authority.

## Decisions

1. **Replace `detectCyrillicLanguage` with `classifyBySnippet(text, candidates)`** — a synchronous, candidate-set-scoped set-difference classifier; sole authority for per-snippet concealment.
2. **Each language ships a declarative `LanguageProfile = { code, alphabet, words: { function, frequent } }`.** No per-character weights, no `scriptDefault`, no coverage metric. `alphabet` and `words.function` are hand-auditable; `words.frequent` is corpus-generated.
3. **Distinctiveness is candidate-set-relative and computed by set-difference** at increasing granularity: a signal is distinctive for X iff it's in X's profile and in **no other candidate's**. Worked example (chars): `і` is uk-vs-ru distinctive but _not_ uk-vs-be (both have `і`); `ы` is ru-vs-uk distinctive but **inert in `{be,ru}`** (be uses `ы`). At word granularity the same logic separates nested-alphabet pairs (kk←ru) and shared-letter words (`та` = uk "and" is all shared letters, yet a clean uk marker).
4. **Ladder; first confident rung wins; else `unknown → keep`:** rung 1 = alphabet; rung 2a = `words.function` (curated grammatical markers — conjunctions, prepositions, pronouns, particles); rung 2b = `words.frequent` (corpus content words); rung 3 = `franc(text, { only: [candidates] })` (sync inline, gated by a length floor, residual only). No frequency-blind fallback; `unknown` always means "do not conceal."
5. **Rung 3 (franc) is asymmetric and bounded** — runs only when rungs 1–2 abstain (cost confined to the residual); may _add_ a hide on strong evidence (length ≫ `MIN_LENGTH`, high margin) but never strips protection on weak evidence.
6. **Hide model = allowlist baseline + curated imposed overlay.** Conceal iff `verdict.language !== 'unknown'` AND `verdict.language ∉ enabled` AND `verdict.margin ≥ minHideMargin(verdict.rung)` — the block-only asymmetry lives in the predicate: a _keep_ is free, only a _hide_ is gated, and the bar tightens for less-trusted rungs. `enabled = new Set(settings.priority)`. `blocked` is derived = `((⋃ IMPOSED_OVER[n]) ∪ ['ru']) \ enabled`. Only `ru` is unconditionally locked; other imposers are user-overridable.
7. **franc also runs off the hot path as a shadow oracle** (idle, batched, `only:[candidates]`) verifying rung-1/2 decisions; trichotomy confirm / contradict / abstain; only a confident-vs-confident contradiction records a divergence.
8. **Margin (top1 − top2), not franc's normalized top score, is confidence** — for both the classifier and the oracle ([franc-min.ts:91-92](../packages/lang-detect/src/engines/franc-min.ts:91)).
9. **A divergence is a diagnostic signal, not an error** — its own type at info/warn on a channel separate from [`error-boundary`](../apps/extension/src/lib/error-boundary.tsx); never `console.error`.
10. **On-device only.** Diagnostics (incl. the disputed snippet text) live in an ephemeral ring buffer + console; never networked, never synced. [`CorrectionEvent`](../packages/shared/src/index.ts:107) stays content-free. Mirrors [error-boundary.tsx:36-37](../apps/extension/src/lib/error-boundary.tsx:36).
11. **v1 surface = console + ring buffer + optional popup list**, each record copy-paste-ready as a `fixtures.ts` entry. A `chrome.devtools.panels` panel is deferred.
12. **Confirmed divergences graduate to the corpus** in [`fixtures.ts`](../packages/lang-detect/test/fixtures.ts).
13. **The oracle (and rung-3 decision logging) is gated behind a diagnostics flag and/or sampling.**

## Data model

```ts
// packages/lang-detect/src/profiles.ts
import type { LanguageCode } from './engine'; // lang-detect's own alias — keep the package free of @movar/shared

export interface LanguageProfile {
  code: LanguageCode;
  /** Rung 1. The alphabet, lowercased. Distinctiveness is a runtime
   *  set-difference vs other candidates — no precomputed weights, so the same
   *  letter can be decisive in one pair and inert in another (decision 3). */
  alphabet: string;
  words: {
    /** Rung 2a. Curated grammatical markers (conjunctions, prepositions,
     *  pronouns, particles) — small, hand-auditable, highest precision.
     *  Short + closed-class + frequent → reliable on short titles. Hide bar 1. */
    function: readonly string[];
    /** Rung 2b. ~300–500 commonest words, corpus-derived (title/subtitle
     *  register so content words rank in). Recovers distinctive-free text and
     *  nested-alphabet pairs (kk⊇ru) that rung 1 misses. Hide bar 2. */
    frequent: readonly string[];
  };
}

export interface SnippetVerdict {
  language: LanguageCode | 'unknown'; // 'unknown' = do not conceal
  margin: number; // winner − runner-up, normalized 0..1
  rung: 1 | '2a' | '2b' | 3 | null; // which rung decided; for telemetry + divergences
}

/** Sync. Candidates = the user's enabled languages ∪ imposed overlay. */
export function classifyBySnippet(
  text: string,
  candidates: readonly LanguageProfile[],
): SnippetVerdict;
```

### The ladder

For `text` and a candidate set, take the first rung that clears its margin; else `unknown`:

1. **Rung 1 — alphabet.** Attribute each char to a candidate iff it's in exactly one candidate's `alphabet`; tally.
2. **Rung 2a — function words.** Tokenize (split on non-letters, lowercase, keep single-char tokens). Attribute each token iff in exactly one candidate's `words.function`; tally. (Highest precision → lowest hide bar.)
3. **Rung 2b — frequent words.** Same against `words.frequent`.
4. **Rung 3 — franc (gated).** Only if rungs 1–2 abstain _and_ `text.length ≥` the floor: `franc(text, { only: candidates.map(to639_3) })` (sync), mapped back via the franc-min engine's `ISO_639_3_TO_BCP_47` table, accepted only above the franc margin. Asymmetric (decision 5).

Tokenization is why the word rungs see what the alphabet can't: the standalone token `и` is purely Russian (Ukrainian never writes `и` as a word — it's `і`/`та`/`й`), yet the _letter_ `и` is shared (uk `ти`, `син`), so rung 1 must ignore it and rung 2a catches it.

### Worked examples

| Input                                | Rung 1 (alphabet)                    | Rung 2 (words)                                 | Result                                   |
| ------------------------------------ | ------------------------------------ | ---------------------------------------------- | ---------------------------------------- |
| "Слава Україні" (uk)                 | `і` ∈ uk\\{ru} → uk                  | —                                              | **uk**, rung 1                           |
| standalone `и` / `та` tokens         | shared letters → abstain             | `и` ∈ ru.function / `та` ∈ uk.function         | **ru / uk**, rung 2a                     |
| "Доброе утро" (ru, distinctive-free) | abstain                              | `утро`/`доброе` ∈ ru.frequent                  | **ru**, rung 2b (recall recovered)       |
| "Нова музика" (uk, distinctive-free) | abstain                              | `музика`/`нова` ∈ uk.frequent (ru is `музыка`) | **uk**, rung 2b (false positive _fixed_) |
| Kazakh text, `{kk, ru}`              | kk alphabet ⊇ ru → nothing for ru    | disjoint vocabulary decides                    | rung 2 (impossible at rung 1)            |
| "Apple Music" on a uk page           | Latin ∉ Cyrillic alphabets → abstain | in no candidate's words → abstain              | **unknown → keep**                       |

The `be-pure` fixture ([fixtures.ts:80-83](../packages/lang-detect/test/fixtures.ts)) confirms rung 1: `і` throughout, `ы` in "в**ы**става", **no** `и` — so `і`/`и` separates be from ru while `ы` is shared (inert in `{be,ru}`).

### Rung thresholds (conservative starts; oracle-calibrated)

`classifyBySnippet` stays symmetric; the block-only asymmetry lives in the conceal predicate. Shipped strict, loosened from the oracle's measured per-pair false-positive vs. leak rates:

| Rung                | Verdict fires when                    | `minHideMargin` (to hide)                | Length gate     |
| ------------------- | ------------------------------------- | ---------------------------------------- | --------------- |
| 1 — alphabet        | winner ≥1 distinctive char, margin ≥1 | 1                                        | none            |
| 2a — function words | winner ≥1 marker, margin ≥1           | **1** (closed-class → highest precision) | none            |
| 2b — frequent words | winner ≥1 marker, margin ≥1           | **2** (resist single-word contamination) | none            |
| 3 — franc           | `franc()` non-`und`, margin ≥ bar     | hide-only (asymmetric)                   | **≥ ~24 chars** |

Rung 2a's bar of 1 is justified because grammatical markers are closed-class (no borrowing, no proper-noun collisions) and short (present in short titles). Rung 2b's bar of 2 guards against a single contaminating marker; cognates spelled identically cancel via set-difference, and the oracle surfaces systematic false-markers to fix. The integers are not load-bearing — the oracle calibrates them.

## Package boundaries

`@movar/lang-detect` is today a self-contained detection package (deps: `franc-min` only; it defines its own `LanguageCode = string` in [`engine.ts`](../packages/lang-detect/src/engine.ts) and does **not** import `@movar/shared`). The new detection core folds into it and preserves that independence. Three-way split:

- **`@movar/lang-detect`** — pure detection mechanics; no DOM, no `chrome`, no product policy. `LanguageProfile`, `SnippetVerdict`, `classifyBySnippet` (`profiles.ts`); the per-language profile data (`profiles/uk.ts`, …) + the `gen-word-profiles` build script (`packages/lang-detect/scripts/`); the corpus (`test/fixtures.ts`); the pure oracle comparison `classifyDivergence` + `DivergenceKind` (`shadow.ts`). Replaces `detectCyrillicLanguage` _within_ the package; uses lang-detect's own `LanguageCode`, **not** `@movar/shared`.
- **`@movar/shared`** — settings + policy. The `IMPOSED_OVER` native→imposed map and `deriveBlocked(priority)` (the evolution of `enforceLockedLanguages` / `LOCKED_BLOCKED_LANGUAGES`). This is product policy — "which language is imposed over which, and what's locked" — consumed by the redirect layer too, not only conceal. The classifier stays policy-agnostic; shared owns "what should movar do about it?".
- **`apps/extension`** — wiring + DOM/`chrome` plumbing. `conceal.ts` orchestrates settings → candidate set (`enabled ∪ overlay`) → `classifyBySnippet` → predicate → conceal; plus the diagnostics a pure package can't host — the `DetectionDivergence` _record_ (detection result + page context: domain, sample, lengthBucket), the ring buffer, the `console`/`requestIdleCallback` scheduling, and the popup surface.

Dependency direction stays acyclic: `lang-detect` (standalone) ← `apps/extension` → `@movar/shared`; shared does **not** import lang-detect (verified). The two `LanguageCode = string` aliases are structurally identical, so codes pass freely; unifying them is optional cleanup, not worth coupling the packages for.

## Conceal predicate: allowlist baseline + imposed overlay

Replaces the `ru`-only gate at [conceal.ts:176](../apps/extension/src/lib/page-content/conceal.ts:176).

- **`enabled = new Set(settings.priority)`** — reuse `priority`; already the picker keep-set ([content.ts:427](../apps/extension/src/entrypoints/content.ts:427), [filter.ts:456](../apps/extension/src/lib/lang-pickers/filter.ts:456)), user-editable, non-empty, locked-excluded. Order matters for redirect; conceal needs only membership.
- **Predicate:** `verdict = classifyBySnippet(text, candidates)`; **conceal iff `language !== 'unknown'` AND `language ∉ enabled` AND `margin ≥ minHideMargin(rung)`.** Uncertain → kept (safe; conceal is card-scoped). A sub-bar verdict is still recorded for the oracle even though the card stays kept.
- **Imposed overlay:** `IMPOSED_OVER: Record<LanguageCode /*native*/, { imposed: LanguageCode }[]>` in `@movar/shared`, e.g. `{ uk: [{imposed:'ru'}], be: [{imposed:'ru'}], kk: [{imposed:'ru'}], ca: [{imposed:'es'}], gl: [{imposed:'es'}] }`. `candidates = profiles(enabled ∪ ⋃ IMPOSED_OVER[enabled])`. Derived `blocked = ((⋃ IMPOSED_OVER[n].imposed) ∪ ['ru']) \ enabled`; both filters keep reading `settings.blocked`; `enforceLockedLanguages` → `deriveBlocked(priority)`.
- **Lock = `ru` only, unconditional.** Other imposers are derived-into-`blocked` (and bite only when `contentModification` is on) but **overridable** — a bilingual Catalan/Spanish speaker can enable `es`. `ru` alone is movar's non-negotiable and can never enter `enabled`.

## Shadow oracle and rung-3 backstop

franc plays two `only:[candidates]`-scoped roles: **rung 3** (inline, sync — `franc()` itself is synchronous, so no async restructuring — decides the long-enough distinctive-free residual, asymmetric, bounded), and **shadow oracle** (off-path, async, idle, batched — verifies rung-1/2 _decisions_ to catch wrong profile data).

```ts
export type DivergenceKind = 'confirm' | 'contradict' | 'abstain';

export interface DetectionDivergence {
  timestamp: number;
  domain: string; // domain only — CorrectionEvent privacy rule
  candidates: readonly LanguageCode[];
  classifier: {
    language: LanguageCode | 'unknown';
    margin: number;
    rung: 1 | '2a' | '2b' | 3 | null;
  };
  oracle: { language: LanguageCode | null; margin: number; engine: string };
  sample: string; // local-only, ephemeral, trimmed. Never sent.
  lengthBucket: 'xs' | 's' | 'm' | 'l';
}
```

Oracle flow: classifier decides + conceals (sync); concealed/checked snippets push to a bounded queue; on `requestIdleCallback` (gated by flag/sampling) the queue drains through one batched franc pass per candidate scope; `confirm`/`abstain` are no-ops, `contradict` pushes a `DetectionDivergence` to the ring buffer and emits one `console.groupCollapsed('[movar:detect] divergence', …)`.

## Diagnostics surface & privacy

Structured console (`[movar:detect]`, both verdicts + trimmed sample + paste-ready fixture); an in-memory ring buffer (last ~200) on `window.__movar?.diagnostics`; an optional popup list with "copy as fixture". Hard boundary (decision 10): the snippet `sample` exists only in the ring buffer and console — never `chrome.storage`, never a `CorrectionEvent`, never the network.

## Migration / sequencing

Deliberate behaviour change (the frequency-blind fallback is removed; the allowlist predicate replaces the `ru`-gate). Characterize at the **conceal** level.

- **Phase 1 — rungs 1–2 + profiles.** Add `LanguageProfile`, build `uk`/`ru`/`be`/`bg`/`en` profiles (alphabet + curated `words.function` + generated `words.frequent`), and `classifyBySnippet` (rungs 1, 2a, 2b). Distinctive-present cards match today; distinctive-free cards resolve via rung 2 (better) or `unknown → keep`. Tests assert the new behaviour, incl. "Нова музика"/"Доброе утро" and the `та`/`и` function-word pair.
- **Phase 2 — allowlist predicate + overlay.** Switch `conceal.ts`; derive `blocked` from `priority` via `IMPOSED_OVER`; scope candidates to `enabled ∪ overlay`.
- **Phase 3 — rung 3 + shadow oracle.** Gated inline franc; off-path oracle, ring buffer, console; behind the flag.
- **Phase 4 — popup diagnostics + copy-as-fixture.**

## Test strategy

- **Rung 1:** candidate-relative cases (`ы` decides `{uk,ru}`, inert in `{be,ru}`; `і`/`и` decides be↔ru); distinctive-present parity with today.
- **Rung 2a:** `та`/`і`/`й` → uk, standalone `и`/`что` → ru; shared function words (`в`,`на`,`не`) cancel; a lone `и` cannot flip a uk-marker-majority title (margin).
- **Rung 2b:** distinctive-free recall ("Доброе утро"→ru); precision fix ("Нова музика"→uk); kk←ru rescue; identical cognates cancel.
- **Rung 3:** fires only on residual + above the floor; asymmetric; `only:[candidates]`; abstains on short text.
- **Predicate:** confident non-enabled → hidden; `unknown`/sub-bar → kept; LOCKED `ru` can't be enabled away; derived `blocked` correct.
- **Oracle:** trichotomy; franc-`null` → abstain; privacy (sample never persisted; `CorrectionEvent` content-free; no network).
- **Hot-path budget:** per-node loop sync; rung 3 only on residual; oracle only under idle/flag.

## `words` data generation

- **Source:** `words.frequent` from a register-matched, openly-licensed frequency list (OpenSubtitles-derived `FrequencyWords`; subtitle register ≈ titles; broad coverage incl. uk/ru/be/kk/ca/gl — verify license per language), via a checked-in `scripts/gen-word-profiles.mts`; **generated arrays committed, never fetched at runtime**. `words.function` is hand-curated and reviewed.
- **Size:** `words.frequent` ~300–500/language to start (register-matched so content words rank in), calibrated per language from the oracle's leak rate. ~400 × ~10 langs × ~8 B ≈ ~32 KB.
- **Contamination:** raw per-language lists; **set-difference at runtime** (distinctiveness is candidate-set-relative — can't pre-difference); `≥2`-marker bar (2b) + oracle loop catch asymmetric false-markers → fix by editing a list or a small shared-exclusion set. Normalize: lowercase + strip non-letters; **no diacritic folding** (diacritics are the signal for Latin pairs).

## Risks

- **Word-list maintenance & size** — `words.frequent` is corpus-generated and low-KB; stale/short lists weaken rung 2b (→ more `unknown → keep`, never a false hide). `words.function` is small but must be reviewed by a speaker.
- **Proper-noun / numeric titles** hit no marker → rung 3 or `unknown → keep`. Measurable via the oracle.
- **Rung-3 inline cost** bounded to the residual; cap per-tick rung-3 calls if a mostly-distinctive-free page pushes many nodes to franc.
- **Uneven franc coverage** (~45 mapped codes) → unmapped languages always abstain at rung 3. Surface per-pair abstain rate.
- **Diagnostics noise** if margins too low. Start strict; tune from data.

## Considered alternatives

- **Weighted-character profiles (CLDR weights + `scriptDefault` + coverage).** Rejected — needs non-auditable frequency data, can't help distinctive-free text, can't separate nested-alphabet pairs. The set-difference ladder is simpler and strictly more capable.
- **Pure alphabet, no `words` rung.** Rejected — regresses `uk←ru` recall and is impossible for `kk←ru`.
- **Single flat `words: string[]` (no function/frequent split).** Rejected — function words are higher-precision and short-title-friendly, justifying a lower hide bar; the split also encodes the bar structurally and separates curated vs generated provenance.
- **franc-min as the per-snippet authority.** Rejected — short-text unreliability, false positives, coverage, hot-path cost.
- **Pure allowlist, drop the overlay.** Rejected — loses anti-imposition identity, distinctive-free recall, and the LOCKED guarantee.
- **Keep the bespoke cascade / its frequency-blind fallback.** Rejected — over-hides native distinctive-free titles; doesn't scale.
- **CLD3 / ELD / multi-engine consensus on the action path.** Heavier; revisit only if franc abstains too often as a backstop.
- **Networked divergence telemetry.** Rejected — violates the on-device invariant.

## Open questions

- **Resolved this review:** detection model (ladder), data shape (`{ alphabet, words: { function, frequent } }`), hide model (allowlist + overlay), config (`enabled = set(priority)`, derived `blocked`), thresholds, `words` source/size/contamination. Remaining: concrete threshold integers (oracle-calibrated post-launch).
- **`enabled` vs `priority` divergence** — reuse `priority` (v1) or a distinct `enabled` list for feed-keep ≠ redirect-prefer, if demand appears.
- **Orthographic-mark evidence (parked 2026-06-03, polish alongside Phase 3).** uk/be use an intra-word apostrophe (`комп'ютер`, `сім'я`) where ru uses `ъ`/nothing — the uk/be mirror of ru's already-captured `ъ`. Model as a small `marks` field merged into the rung-1 character set (uk/be → `'’ʼ`), or, for precision, a regex feature matching `Cyrillic ' Cyrillic`. Dominant-script scoping already prevents the English-contraction collision. Low conceal-layer impact — it only ever argues for uk/be → _keep_, never for hiding ru — so it's a classifier-correctness / shadow-oracle-signal polish, not a recall lever. The regex form is the seed of a general orthographic-features mechanism if a second feature appears.
